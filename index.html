VISTAS en MySQL

Las VISTAS en MySQL son como tablas virtuales que te permiten visualizar datos de una o más tablas subyacentes. Se crean a partir de una consulta SELECT y actúan como una capa intermedia entre los usuarios y las tablas reales.

×

Ejemplo de creación: CREATE VIEW Vista_Correos AS SELECT nombres, apellidos, correo FROM users;

Ejemplo de uso: SELECT * FROM Vista_Correos;

Ejemplo de eliminación: DROP VIEW Vista_Correos;

×

Ejemplo de creación: CREATE VIEW persona_gusta_merengue AS SELECT nombres, apellidos, musica FROM users WHERE musica = 'merengue';

Ejemplo de uso: SELECT * FROM persona_gusta_merengue;

×

Ejemplo de creación: CREATE VIEW persona_llamada_felipe AS SELECT nombres, apellidos FROM users WHERE nombres LIKE '%Felipe%';

Ejemplo de uso: SELECT * FROM persona_llamada_felipe;

×

Ejemplo de creación: CREATE VIEW persona_24_años AS SELECT nombres, apellidos, edad FROM users WHERE edad = 24;

Ejemplo de uso: SELECT * FROM persona_24_años;

×

Ejemplo de creación: CREATE VIEW personas_de_Ecuador_o_Venezuela AS SELECT nombres, apellidos, pais FROM users WHERE pais = 'Ecuador' OR pais = 'Venezuela';

Ejemplo de uso: SELECT * FROM personas_de_Ecuador_o_Venezuela;

Procedimientos Almacenados en MySQL

Los procedimientos almacenados son programas que se almacenan en la base de datos y se pueden ejecutar desde una consulta SQL o desde otro procedimiento almacenado. Permiten automatizar tareas complejas y mejorar el rendimiento de las consultas.

×

Obtener datos básicos

Este procedimiento devuelve los nombres y apellidos de los usuarios:

CREATE PROCEDURE obtener_datos_basicos()
BEGIN
    SELECT nombres, apellidos FROM users;
END //

Ejecución: CALL obtener_datos_basicos;

×

Obtener edad y país

Este procedimiento devuelve la edad y el país de los usuarios:

CREATE PROCEDURE obtener_edad_y_pais()
BEGIN
    SELECT edad, pais FROM users;
END //

Ejecución: CALL obtener_edad_y_pais;

×

Personas llamadas Felipe

Este procedimiento devuelve los nombres y apellidos de las personas cuyo nombre es Felipe:

CREATE PROCEDURE persona_llamada_felipe()
BEGIN
    SELECT nombres, apellidos FROM users WHERE nombres LIKE "%Felipe%";
END //

Ejecución: CALL persona_llamada_felipe;

×

Personas mayores de 50 años y que les gusta el vallenato

Este procedimiento devuelve los nombres, edades y géneros musicales de las personas mayores de 50 años que les gusta el vallenato:

CREATE PROCEDURE persona_mayor_50_y_gusta_vallenato()
BEGIN
    SELECT nombres, edad, musica FROM users WHERE musica = "vallenato" AND edad >= 50;
END //

Ejecución: CALL persona_mayor_50_y_gusta_vallenato;

×

Nombres que no se repiten

Este procedimiento devuelve los nombres que no se repiten en la tabla de usuarios:

CREATE PROCEDURE nombres_que_no_se_repiten()
BEGIN
    SELECT DISTINCT nombres FROM users;
END //

Ejecución: CALL nombres_que_no_se_repiten;

Diferencia entre SQL y NoSQL

SQL se basa en un modelo relacional con tablas estructuradas y un esquema fijo. Es bueno para transacciones complejas y garantiza la consistencia. NoSQL, por otro lado, ofrece flexibilidad en el esquema y escalabilidad horizontal.

¿Qué es MongoDB?

MongoDB es un sistema de gestión de bases de datos NoSQL que almacena datos en documentos flexibles similares a JSON. Ofrece escalabilidad horizontal, alto rendimiento y consultas potentes.

Operaciones básicas en MongoDB

Listar

Ejemplo de listar: db.empleados.find({departamento: "Ventas"})

Actualizar

Ejemplo de actualizar: db.empleados.updateOne({_id: "12345"}, {$set: {salario: 2000}})

Eliminar

Ejemplo de eliminar: db.empleados.deleteOne({_id: "12345"})

Inserción

Ejemplo de inserción: db.empleados.insertOne({nombre: "Juan", apellido: "Pérez", departamento: "Ventas"})
